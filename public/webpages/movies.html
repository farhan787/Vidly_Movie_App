Movies API Endpoint

About

This endpoint performs operations on the movies stored in the MongoDB database. You have to send your requests to /api/movies. It responds to 4 HTTP methods, GET, POST, PUT and DELETE.

Visit Code

GET  /api/movies

A GET request to this endpoint, returns a list of all available movies stored in the MongoDB database.

GET with movie id  /api/movies/id

A GET request with the movie id to this endpoint, returns a movie that belongs to the movie id embedded in the url movieId.

POST   /api/movies

We can post a movie if we are authenticated user via POST request, Send title, genreId, numberInStock, dailyRentalRate in the body of the request, also send x-auth-token in the request header and the value for this x-auth-token will be the authentication token, that we get from the POST method of Auth endpoint.

{
 "title": "movie title",
 "genreId": "5e81f584f5d05b001666f37f",
 "numberInStock": "100",
 "dailyRentalRate": "100",
}

PUT   /api/movies/id

If we are authenticated user, we can update the movie stored in the database via PUT request by sending the new body of movie along with the x-auth-token in the request header as discussed in POST request.

{
 "title": "updated movie title",
 "genreId": "updated genre id",
 "numberInStock": "updated stocks",
 "dailyRentalRate": "updated price",
}

DELETE   /api/movie/id

If we are authenticated user and admin, we can delete a movie stored in the database via DELETE request along with the movie id in the url.
