Genres API Endpoint

About

This endpoint performs operations on the genres stored in the MongoDB database. You have to send your requests to /api/genres. It responds to 4 HTTP methods, GET, POST, PUT and DELETE.

Visit Code

GET  /api/genres

A GET request to this endpoint, returns a list of all available genres stored in the MongoDB database.

GET with genre id  /api/genres/id

A GET request with the genre id to this endpoint, returns a genre that belongs to the genre id embedded in the url genreId.

POST   /api/genres

We can post a genre if we are authenticated user via POST request, send name in the body of the request, also send x-auth-token in the request header and the value for this x-auth-token will be the authentication token, that we get from the POST method of Auth endpoint.

{
 "name": "new genre category",
}

PUT   /api/genres/id

If we are authenticated user, we can update the genre stored in the database via PUT request by sending the new body of genre along with the x-auth-token in the request header as discussed in POST request.

{
 "name": "updated genre name",
}

DELETE   /api/genres/id

If we are authenticated user and admin, we can delete the genre stored in the database via DELETE request along with the genre id in the url.
