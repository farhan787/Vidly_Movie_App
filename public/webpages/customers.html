Customers API Endpoint

About

This endpoint performs operations on the customers stored in the MongoDB database. You have to send your requests to vidly-app.herokuapp.com/api/customers. It responds to 4 HTTP methods, GET, POST, PUT and DELETE.

Visit Code

GET  /api/customers

A GET request to this endpoint, returns a list of all the customers stored in the MongoDB database.

GET with id  /api/customers/customerId

A GET request with the customer id to this endpoint, returns a customer that belongs to the customer id embedded in the url customerId.

POST   /api/customers

We can post a customer if we are authenticated user via POST request, send name and phone in the body of the request, also send x-auth-token in the request header and the value for this x-auth-token will be the authentication token, that we get from the POST method of Auth endpoint.

PUT   /api/customers/customerId

If we are authenticated user, we can update the customer information stored in the database via PUT request by sending the new body of customer along with the x-auth-token in the request header as discussed in POST request.

DELETE   /api/customers/customerId

If we are authenticated user and admin, we can delete a customer stored in the database via DELETE request along with the customer id in the url.
